## 🗺️ Описание графа навигации

Данный проект использует систему графов для построения маршрутов внутри зданий. Логика навигации строится на взаимодействии двух слоев данных: узлов (Nodes) и рёбер (Edges).

### 📍 Узлы (Nodes)

Узлы представляют собой конкретные точки на карте этажа. Каждый узел обладает следующими обязательными атрибутами:
* **name** (string): Уникальный идентификатор узла. Для кабинетов это их номер, для технических узлов — уникальное системное имя.
* **floor** (int): Индекс этажа (например, 0, 1, 2).
* **type** (string): Категория объекта.

#### Типы узлов:
1. **Room** — кабинеты и аудитории. Поле `name` содержит номер кабинета. Это конечные точки маршрутов.
2. **Hallway** — технические узлы в коридорах. Используются для обозначения поворотов и разветвлений графа. Их названия (`name`) не несут смысловой нагрузки для пользователя, но обязаны быть уникальными.
3. **Stairs** — лестничные площадки. Точки входа на лестницу для перемещения между этажами.
4. **Elevator** — точки входа в лифт.

---

### 🛣️ Рёбра (Edges)

Рёбра соединяют узлы между собой, определяя доступные пути перемещения. Атрибуты ребра:
* **floor** (int): Этаж, на котором находится данный путь.
* **type** (string): Тип прохода.

#### Типы рёбер:
1. **Room** — соединительные пути, которые ведут к кабинетам. Связывают один или два узла типа Room с дорожной сетью.
2. **Corridor** — основные пути перемещения, соединяющие узлы типа Hallway (коридорные повороты).
3. **Stairs** — вертикальные связи. Используются для соединения узла типа Stairs с аналогичным узлом на другом этаже.
4. **Elevators** — вертикальные связи через лифтовые шахты, соединяющие узлы типа Elevator.

---

### 🛠️ Технические особенности
* **Координатная сетка:** Все геометрические данные хранятся в проекции **EPSG:3857**, что обеспечивает корректное отображение на веб-картах.
* **Уникальность:** Уникальность поля `name` у всех типов узлов является критическим требованием для корректной работы алгоритма поиска пути (Dijkstra) и связи рёбер в базе данных.
* **Автоматизация:** Скрипты импорта (`seed.py`) автоматически обрабатывают GeoJSON файлы, создавая двусторонние связи для навигации и связывая лестницы/лифты между соседними этажами на основе пространственной близости.

---

### Как обновить карту
1. Подготовьте файлы `nodes.geojson` и `edges.geojson`.
2. Убедитесь, что атрибуты `floor` и `type` заполнены для каждого объекта.
3. Поместите файлы в директорию `backend/data/campus/` и запустите скрипт миграции данных.
